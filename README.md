# お手伝いクエスト（homework-quest）

宿題・お手伝いを自分でチェックして進める一人用アプリ。
達成すると **ポイント → レベルアップ／バッジ** で達成感が出て、頑張りは
**カレンダー・連続日数（ストリーク）・グラフ** で見える化します。

**公開URL**: <https://yu-ichi7.github.io/homework-quest/>
（GitHub Pages 配信。PCやサーバー不要で、いつでもどこからでも開けます）

- 一人用。チェックしたら即ポイント確定（承認フローなし）
- 定番ルーティン（毎週この曜日）＋スポット（この日だけ）の両対応
- **週・月ごとの目標**を立てられる（ポイント目標は自動進捗、自由記述は手動チェック）。
  達成するとボーナスコイン
- 貯めたポイントは **コイン** として消費でき、レトロドット風の**冒険マップ**を進める
  ミニRPG要素あり（バトルなし。宝箱・装備集め中心）
- サーバーレスの静的サイト。**データは端末内（localStorage）にのみ保存**され、
  どこにも送信されない

## スマホで使う

1. スマホのブラウザで公開URLを開く
2. **ホーム画面に追加**（iOS: 共有ボタン→「ホーム画面に追加」/ Android: メニュー→「ホーム画面に追加」）
3. 以後はホーム画面のアイコンからアプリのように起動できる

> iOSのSafariは、長期間使っていないサイトの保存データを消すことがあります。
> ホーム画面に追加して使えばこの対象になりにくく、念のため下記のバックアップも使えます。

## データとバックアップ

- 記録は**その端末の中だけ**に保存されます（端末ごとに独立）
- 「設定」画面の **書き出し（保存）** でJSONファイルとしてバックアップ、
  **読み込み（復元）** で戻せます。機種変更のときはこの書き出し→読み込みで引っ越し
- データを初期状態に戻したいときは、ブラウザのサイトデータ削除（またはバックアップを読み込み）

## 画面

- **今日**（`index.html`）… 今日のタスク。タップで達成・ポイント加算、レベルバー、バッジ、お祝い演出、目標の要約
- **記録**（`record.html`）… ストリーク・週間/月間集計・棒グラフ・カレンダー・目標の進捗
- **冒険**（`adventure.html`）… コインを使って冒険マップを進める。宝箱・装備・おみせ
- **設定**（`parent.html`）… タスクの追加・削除、名前・色の編集、週/月の目標登録、バックアップ

## ポイントとコインの関係

- **XP**（レベル・バッジ用）… タスク達成の累計。減ることはない
- **コイン**（冒険で使う）… タスク達成・目標ボーナスで貯まり、冒険マップの移動や
  おみせでの購入で**消費される**。チェックを取り消すと、獲得ぶんのコインも戻る

## 開発

```bash
npm install
npm run dev        # ローカル確認用サーバー → http://localhost:3100
```

本番は GitHub Pages が `docs/` をそのまま配信する（ビルド不要）。
`master` に push すれば数分で公開に反映される。

### 構成

- `docs/` … 公開される静的サイト本体
  - `js/lib/` … ロジック（レベル計算・バッジ判定・ストリーク・タスク展開など、純粋関数）
  - `js/store.js` … localStorage への保存・読み出し（旧サーバーAPIと同じ意味論）
  - `sw.js` … Service Worker（network-first。圏外でも一度開いたページは開ける）
  - `manifest.json` … ホーム画面追加用
- `dev-server.js` … ローカル確認用の静的サーバー（本番では使わない）

### 設計メモ

- XP/レベル/バッジは達成ログ（completions）を「正」として毎回再計算するため、
  チェック・取り消しでズレが起きない
- 日付はすべて端末のローカルタイム基準（YYYY-MM-DD文字列）
